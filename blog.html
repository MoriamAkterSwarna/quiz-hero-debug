Map VS Filter VS Find VS ForEach

Map: The function.map() runs the exact same code on each element of an array and then returns a new array containing the changed elements.

Filter: The function.filter() checks each element in an array to discover if it matches a list of requirements and returns a new array with the elements that do.

Find: The function .find() checks the elements in an array that the conditions is true or not. It returns the first element of the matched array.

ForEach: The function.forEach() runs the similar code on each element of an array, but it does not alter the array and returns undefined.

Cookies VS Local Storage VS Session Storage

Cookies: Cookies expiry depends on the setting and working per window. It supports more older browser. It has 4KB storage and it provides server and client side reading.

Local Storage: It is not session based rather it needs to be deleted via JS or manually. It supports less older browser. It has 5MB/10MB storage and it provides client side reading only.

Session Storage: It is session based and working per window or tab. It supports less older browser. It has 5MB storage and it provides client side reading only.

What is Event Loop? What is the purpose of Event Loop?

Event Loop: The runtime model of JavaScript is based on an event loop, which executes the code, gathers and processes events, and performs queued sub-tasks.
The Call Stack and the Callback Queue are two simple tasks performed by the Event Loop. The Event Loop will pull the first event from the queue and put it to the Call Stack if it is empty, where it is effectively run.
